# Documentation

The `docs` module contains product, technical, UX, data, AI, and API specifications for IPMS.

## Folder Purpose

- `vision.md`: Product vision, target users, goals, and success criteria.
- `requirements.md`: Functional and non-functional requirements.
- `user-flows.md`: Role-based flows for students, supervisors, and administrators.
- `ui-specification.md`: Frontend interaction model, screens, components, and visual rules.
- `database-design.md`: Data model, relationships, constraints, and data lifecycle.
- `ai-engine.md`: AI capabilities, prompts, scoring logic, model integration, and evaluation approach.
- `api-specification.md`: Backend API contracts, authentication model, request/response formats, and error conventions.

## Documentation Rules

- Keep architecture decisions close to the relevant document.
- Update docs before or alongside implementation changes that alter behavior.
- Treat these documents as living specifications, not final one-time notes.
- Avoid implementation code in this module.
